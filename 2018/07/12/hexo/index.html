Hexo搭建github博客手记

之前写笔记都是用印象笔记，也想过搭建一个博客分享笔记，一直没有做，今天刚好有时间就着手开始实施了，把搭建的步骤记录下来，以备以后参考

首先介绍两个官网

  • hexo官网
  • hexo-next官网

环境配置

  • 下载并安装node (window系统访问node官网下载安装包,mac系统可以使用Homebrew工具进行安装)
  • 安装hexo,使用npm安装hexo(全局安装)，打开cmd运行 $ npm install -g hexo-cli
  • 下载并安装git。(window系统访问git官网下载安装包,mac系统可以使用Homebrew工具进行安装)

本地搭建hexo系统

  • 新建一个文件夹，如hexo
  • 进入该文件里面，用bash命令，在bash中输入: hexo init(生成hexo模板)
  • 模板生成之后，输入命名 npm install
  • 最后运行 hexo server (开启服务，运行该程序，访问本地localhost:4000)

将本地博客与GitHub相关联

  • 登陆自己的github系统，用自己GitHub的用户名创建一个xxx.github.io的仓库，比如，我的给github用户名为Kyseme,则我创建的仓库名名为Kyseme.github.io
  • 如果github没有配置ssh,首先需要配置ssh，我的是已经配置好的，直接google或者百度搜索配置ssh，能找到不少文章

  • 打开hexo文件夹，里面有_config.xml文件，修改配置，在文件末尾部分(注意type冒号和git中间有一个空格，后面的一样有空格)

  • 运行 npm install hexo-deployer-git –-save

  • 运行 hexo g（本地生成静态文件）
  • 运行 hexo d（将本地静态文件推送至Github）
    此时，打开浏览器，访问 http://Kyseme.github.io

更改hexo主题为hexo-next

  • 下载next主题

完成之后，可以看到themes文件夹下面多了一个next文件夹

  • 启用next主题，在hexo的_config.yml中设置

  • 运行 hexo clean (清除缓存文件 (db.json) 和已生成的静态文件 (public)。
    在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令)

  • 运行 hexo g（本地生成静态文件）
  • 运行 hexo d（将本地静态文件推送至Github）

新建文件

可以执行下列命令创建一篇新文章

您可以在命令中指定文章的布局（layout），默认为 post，可以通过修改 _config.yml 中的 default_layout 参数来指定默认布局。

Hexo 有三种默认布局：post、page 和 draft，它们分别对应不同的路径，而您自定义的其他布局和 post 相同，都将储存到 source/_posts 文件夹。

设置阅读全文

在文章中使用< !–more–> 手动进行截断
这种方法可以根据文章的内容，自己在合适的位置添加 < !–more–> 标签，使用灵活，也是Hexo推荐的方法。
